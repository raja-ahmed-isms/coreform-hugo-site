RLM Server activation

Possible problems.

A floating license is shared over the network with a server or network node as the license manager. See Network Activation for information about setting up a floating license on a network.

If you are having problems problems using Coreform Cubit with a floating license, the problem might be that the RLM-Server is not running properly or the computer with Coreform Cubit installed cannot connect to the RLM-Server

  • Troubleshoot the RLM-Server
  • Connect Coreform Cubit to the RLM-Server

Troubleshooting RLM-Server.

Is the RLM-Server running?

In a web browser, type "localhost:5054" or "[server IP address]:5054" to open the RLM Admin interface. If you are not able to connect to localhost:5054, then RLM is not running.

Solution. Restart RLM by navigating to the RLM-Server folder and running "./rlm.sh start" in Linux or "add-rlm.bat" in Windows. Try again to open the Admin interface.

Is the Coreform ISV server running?

Open the RLM Admin page. Click on the Status button in the left column. You should see the license file in the first table. There should be an ISV Server table at the bottom. In the table you should see "Coreform" ISV running on port 5055. If not, the Coreform ISV server is not running.

Solution. If you do not see an ISV Server table, click the Edit License Files button in the left column and check the license file to ensure that their is a line starting with HOST and one starting with ISV on separate lines. Do not change anything in the lines that begin with "LICENSE" or it will break the license.

Next, click the Shutdown button in the left column, shutdown all servers, and then click the Reread/Restart Server button in the left column. Do not reboot your computer.

Click the Status button and look for the ISV Server table with Coreform running on port 5055.

Does the license file have the correct features?

Click the Status button on the left. In the ISV Server table, click the "Coreform" button in the Server Status column. In the License Pool Status table, you should see the products in the license file with the version number and expiration date.

  • cubit
  • cubit_core
  • cubit_fea
  • cubit_cfd
  • cubit_[translator name] (if purchased a translator)

Solution. If you are not seeing the features in the Product column, click the Status button to ensure that the correct license file is being used.

Click the Shutdown button in the left column to shutdown the ISV server, then click the Reread/Restart Server button. Click the Status button, then the "Coreform" button under the Server Status column.

Is the license file a floating license?

Network installations require a floating license of Coreform Cubit. A network product key is used to activate a license on the RLM-Server. If you activate a node-locked license on the server, only the server can access the license. Do not use the network product key to activate Coreform Cubit on the client computer.

Solution. Activate a floating license on the RLM-Server and point each Coreform Cubit installation to the server.

Troubleshooting the connection of Coreform Cubit to RLM-Server.

For network installations, only the server or computer designated as the license manager is activated with a floating license. When launched, each installation of Coreform Cubit checks the license manager for an available Coreform Cubit license.

Coreform Cubit is unable to check out a license from the server.

If a license is available, Coreform Cubit will launch. If a license is not available, Coreform Cubit will indicate that it cannot check out a license from the server.

Solution. In a web browser, type "localhost:5054" or "[server IP address]:5054" to open the RLM Admin interface. Click on the Status button in the left column. In the ISV Server table, click the "Coreform" button in the License Usage column. Click the "Get License Usage" button. If another Coreform Cubit installation is using the license, close Coreform Cubit on that computer.
