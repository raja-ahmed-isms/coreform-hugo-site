License update

Update a license to a new version.

Software updates include downloading the new software version and updating the local license file.

  • Update a node-locked license
  • Update a floating license

Update a node-locked License.

A node-locked license is installed on a single standalone computer, whether or not it is connected to a network.

Note: The local license file needs to be updated when upgrading to a minor release. For example, 20.1 to 20.2. However, the license file does not need to be updated for maintenance releases, such as 16.0.1 or 16.0.2.

To upgrade a node-locked license to a new version,

  1. Download the software update installation package for your computer's operating system from the My Downloads area of your Coreform account.
  2. Install the software on the same computer where the software was previously installed and activated. The software will by default install to a folder with the version number (for example, c:\Program Files\Coreform Cubit 2020.2). Maintenance releases will install into the same folder as the release version. (Cubit 20.2 installs into c:\Program Files\Coreform Cubit 2020.2)
  3. Launch the application.
    • If the license is not valid, the application will not launch and an activation window will open letting you know that no valid license was found. Enter a valid product key and press Activate.

    • If the license is still valid, but needs to be updated, click Product Activation on the Help menu to open the Product Activation window.

  4. In the Product Activation window, enter the product key that is found on the license page under My Licenses and click Activate.

If the license is not activated, an error message will appear in the lower left corner of the window. Possible errors are incorrect permissions to save to the license folder or the application cannot connect to the activation server because of no Internet connection or blocked by a firewall.

Note: A node-locked product key will activate the application for the local computer only. A node-locked application cannot be activates using a floating product key.


Update a floating License.

If the expiration date or some other option of your license changes, the license on your license server must be updated to reflect the change. Updating a floating license involves re-activating the floating license in a similar way to the original activation. You might also need to update the license manager on the server and update the application on computers connected to the network, if newer versions are available.

Updating the RLM-Server.

Although previous versions of RLM-Server should still manage newer application licenses, we recommend that you update the RLM-Server to match the version of the application you are using. For example, a Cubit 20.2 license will run on RLM-Server 20.1, but we recommend updating to RLM-Server 20.2.

To update the RLM-Server:

  1. Uninstall the older license manager. This can be done by running the uninstall utility located in the RLM-Server folder. Several files will remain in the folder after uninstalling RLM-Server, including the license file.
  2. Download the new RLM-Server.
  3. Install the updated RLM-Server software on the server. For more information, see RLM-Server installation instructions.

Updating the floating license on the license server.

Note: The local license file in the RLM-Server folder of the license server must be updated whenever a change is made to the license, such as a change to the expiration date or a version upgrade. However, the license file does not need to be updated for maintenance releases, such as 20.2.1 or 20.2.2.

To update the floating license on the license server,

Windows
  1. Run the rlm_activate utility to open the Product Activation dialog.
  2. On the Product Activation dialog, enter the product key and press Activate. The software license is now updated.

Linux or Mac OS X
  1. From a terminal run as administrator:

    .\rlm_activate --activate <product_key> <hostname>

    where <product_key> is the product key for the license and <hostname> is the name of the server . The hostname is optional, but will identify the server in your list of licenses in your account.

Updating the workstation software to the latest version.

If the application version has changed, you should install the new software version on each computer that will access the license over the network.

To install the new version,

  1. Download the appropriate application from your My Downloads in your Coreform account.
  2. Install the application update on each network computer where the application will be run.
  3. During the installation process, you will be prompted to setup the license. Select Floating: Connect to a license server. Enter the IP address or hostname of the license server, where RLM-Server is installed and activated. Press Connect.

Note: A floating product key should not be used to activate a node-locked license. A floating product key can only be used to activate or update a floating license on the license server where RLM-Server is installed. Conversely, a standalone product key cannot be used to activate a floating license.
